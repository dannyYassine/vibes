## What is the Capstone?

The capstone project is your opportunity to apply everything you've learned in this course to build a real, end-to-end AI system. Unlike the smaller projects in each section, the capstone is a full-stack production-ready application that demonstrates mastery across multiple modules.

## Goals

By completing the capstone, you will:

1. **Integrate multiple techniques** from the course — RAG, agents, prompting, and/or reasoning
2. **Build something production-ready** — proper error handling, logging, APIs, and a UI
3. **Make deliberate architecture decisions** and justify your choices
4. **Evaluate your system** with appropriate metrics
5. **Present and defend** your approach to peers

## Requirements

Your capstone must include:

### Minimum Requirements

- **LLM integration**: Uses at least one LLM through an API or locally (Llama, Ollama, etc.)
- **Backend**: FastAPI (or equivalent) with proper REST API design
- **Frontend**: A functional UI (can be simple, doesn't need to be polished)
- **One advanced technique**: RAG, agents/tool-calling, or fine-tuning
- **Evaluation**: Evidence that your system works — benchmark numbers, test cases, or user study
- **Documentation**: README with architecture, setup, and usage

### Bonus Points

- **Two or more advanced techniques** integrated (e.g., RAG + agents)
- **Production deployment** (Docker, cloud, etc.)
- **Quantitative evaluation** (RAGAS scores, task completion rate, etc.)
- **Novel problem** not covered in course projects

## Suggested Project Categories

### Category A: Information Retrieval Systems
Build a specialized knowledge assistant over a domain-specific corpus:
- Technical documentation Q&A (your company's docs, open source project)
- Research paper summarizer and explorer
- Legal document analysis system
- Medical literature assistant

### Category B: Autonomous Agents
Build an agent that autonomously completes multi-step tasks:
- Code review and improvement agent
- Data analysis agent (uploads CSV, answers questions, generates visualizations)
- Research synthesis agent
- Personal productivity agent

### Category C: Reasoning Systems
Build a system that demonstrates advanced reasoning:
- Math tutoring system with step-by-step explanations
- Debugging assistant that reasons through code errors
- System design interviewer and evaluator
- Business strategy analyzer

### Category D: Multi-modal Applications
Build a system that combines text with images or other modalities:
- Visual question answering over a photo album
- Product description generator from images
- Video content analyzer and tagger
- Document understanding (extract info from scanned PDFs)

## Evaluation Criteria

Your project will be assessed on:

| Criterion | Weight | Description |
|-----------|--------|-------------|
| Functionality | 30% | Does it work reliably? Edge cases handled? |
| Technical depth | 25% | Appropriate use of course techniques |
| Architecture | 20% | Clean design, justified decisions |
| Evaluation | 15% | Evidence of quality (metrics, tests) |
| Presentation | 10% | Clear explanation and demo |

## Key Takeaways

- The capstone integrates multiple course techniques into a complete system
- Focus on one problem solved well rather than many features done poorly
- Documentation and evaluation are as important as the code
- The best projects solve a real problem you personally care about
